## Your OpenCode Workflow

Here is the recommended workflow for daily use:

```text
1. Start with Plan mode for unfamiliar tasks.
2. Switch to Build mode to implement.
3. Use @scout for quick lookups.
4. Use @explore for deep understanding.
5. Use @general for implementation.
6. Use /undo freely — all actions are reversible.
7. Use /compact to manage context.
8. Switch models based on task complexity.
9. Create custom commands for repetitive tasks.
10. Build skills for domain-specific expertise.
```

## Best Practices Summary

### Do This

```text
✓ Plan before you build.
✓ Use cheap models for simple tasks.
✓ Attach only relevant files with @.
✓ Use /compact in long sessions.
✓ Store API keys in environment variables.
✓ Create custom commands for repetitive work.
✓ Use Plan mode for code review.
✓ Use @scout for quick searches.
✓ Share sessions with /share for collaboration.
```

### Avoid This

```text
✗ Do not use expensive models for trivial tasks.
✗ Do not attach entire folders to context.
✗ Do not hardcode API keys in config files.
✗ Do not submit sensitive data to free provider tiers.
✗ Do not keep MCP servers enabled when not needed.
✗ Do not forget to /compact in long sessions.
```
