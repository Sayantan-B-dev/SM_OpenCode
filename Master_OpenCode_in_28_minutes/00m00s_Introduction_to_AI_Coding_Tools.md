# Introduction to AI Coding Tools (0:00 - 0:07)

## What Normal Coding Tools Do

Traditional AI coding tools operate in a **single-task, single-conversation** paradigm. You open a chat window, ask a question, get an answer, and that conversation is isolated from everything else you're working on. Each tool functions as a standalone unit — one tool, one task, one conversation at a time.

### The Limitations of Traditional AI Tools

- **Isolated conversations**: Every chat is a separate silo with no memory of what other tools or sessions are doing
- **Single-threaded workflow**: You can only work on one task sequentially. While waiting for a response, you cannot parallelize work
- **No project-level awareness**: Most tools don't understand your full codebase structure, architecture patterns, or project conventions
- **Context switching overhead**: Moving between tools means re-explaining context, pasting code snippets, and losing your train of thought
- **No persistent memory**: Once you close a conversation, the context is gone. Next session starts from scratch

### How OpenCode Is Different

OpenCode reimagines what an AI coding tool can be. Instead of a simple chatbot interface, it provides:

- **Multi-session parallelism**: Run multiple AI agents simultaneously on the same project, each working on different tasks
- **Persistent project context**: Tools like `AGENTS.md`, `DESIGN.md`, and memory systems ensure every session knows what other sessions are doing
- **LSP integration**: Automatically loads the right Language Server Protocols so the AI understands your code at a deep level
- **Any model, any provider**: Connect Claude, GPT, Gemini, local models, or 75+ providers through Models.dev
- **Multiple interfaces**: Terminal (TUI/CLI), desktop app, IDE extensions, and even mobile via SSH

This fundamental shift from **sequential, single-conversation tools** to **parallel, multi-agent workflows** is what makes OpenCode a game-changer for developers.
