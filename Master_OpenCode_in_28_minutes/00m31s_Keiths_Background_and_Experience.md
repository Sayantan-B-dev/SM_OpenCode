# Keith's Background and Experience (0:31 - 0:54)

This segment covers the background of Keith (the presenter/creator) and his journey in AI-assisted coding. 

**Note for viewers**: This section is primarily biographical context about the speaker's experience with AI coding tools, their frustrations with existing solutions, and what inspired them to create or champion OpenCode. It provides credibility and context but is not critical technical content for using OpenCode itself.

### Key Takeaways (If You Want to Skip Ahead)

- The speaker has extensive experience with various AI coding tools
- Identified the fundamental limitations of single-conversation tools through real-world usage
- This experience directly informed the design philosophy behind OpenCode's multi-agent approach
- If you're already convinced about OpenCode's value, feel free to skip to **01m41s - What is Open Code** for the technical overview

### Why This Background Matters

Understanding the pain points that led to OpenCode helps you appreciate *why* certain design decisions were made:
- The frustration of context-switching between tools → Led to unified multi-session architecture
- The waste of waiting for sequential responses → Led to parallel agent execution
- The loss of context between sessions → Led to persistent memory files like `AGENTS.md`

This is the lived experience of using AI tools at scale and finding them inadequate — the very problem OpenCode was built to solve.
