# Step-by-Step Installation Guide (3:19 - 3:42)

## How to Install OpenCode

The installation process branches into two paths based on your preference:

```
Install OpenCode
       │
       ├── Desktop App (GUI) → Standalone application with visual interface
       │
       └── CLI/Terminal → Command-line tool with TUI interface
```

### Step 1: Visit the Official Website

Go to **[https://opencode.ai](https://opencode.ai)** — this is the central hub for all installation options. The site provides:
- Direct download links for the desktop app
- Terminal installation scripts for all platforms
- Documentation and getting-started guides
- Links to the GitHub repository (160K+ stars)

### Step 2: Choose Your Path

#### Option A: Desktop App (GUI)
- **Best for**: Users who prefer visual interfaces, point-and-click
- **Available on**: macOS, Windows, Linux (beta)
- **Features**: Built-in terminal, file explorer, session management UI
- **Download from**: [https://opencode.ai/download](https://opencode.ai/download)

#### Option B: CLI / Terminal
- **Best for**: Developers who live in the terminal, power users
- **Available on**: macOS, Linux, Windows (via WSL), Arch Linux
- **Features**: Full TUI with keyboard shortcuts, scripting support, maximum flexibility
- **Install via**: curl, npm, brew, scoop, choco, or package managers

### Which Should You Choose?

| Factor | Desktop App | CLI/Terminal |
|--------|-------------|--------------|
| Ease of setup | ★★★★★ | ★★★☆☆ |
| Keyboard workflow | ★★★☆☆ | ★★★★★ |
| Scripting/automation | ★★☆☆☆ | ★★★★★ |
| Resource usage | Higher | Lower |
| Multi-session management | Visual tabs | Root terminal orchestration |
| Mobile access | No | Yes (via SSH) |

**Recommendation**: Start with the desktop app if you're new, then explore the CLI as you get more comfortable with OpenCode's workflow.
